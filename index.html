نظام يفرض على كل سيارة: بطاقة دخول + موقف محدد (عمود/صف/رقم)… بدون عشوائية

هذا المقترح يحوّل المواقف من “فوضى وتنافس” إلى “توجيه إلزامي” يعتمد على حساسات + كاميرات + ذكاء اصطناعي لتحديد المواقف الفارغة، تخصيص موقف لكل سيارة، ومراقبة الالتزام وتطبيق المخالفات تلقائيًا.

لا دخول إلا بالبطاقة تخصيص موقف إلزامي منع موقفين حماية ذوي الإعاقة وكبار السن مخالفات موثقة

النظام يحدد… لا عشوائية
ملاحظة: الأرقام النهائية تُحدد حسب سياسة الجهة ومساحة الموقف وعدد المستخدمين

الملخص التنفيذي (Executive Summary) — صفحة واحدة

أولًا: المشكلة

  • ازدحام داخل المواقف بسبب البحث العشوائي.
  • أخذ موقفين أو الوقوف بشكل خاطئ يقلّل السعة الفعلية.
  • اعتداء على مواقف ذوي الإعاقة وكبار السن.
  • نزاعات ومشاجرات نتيجة العشوائية والتنافس.

ثانيًا: الحل المقترح

  • بوابات دخول/خروج إلزامية بالبطاقة.
  • الذكاء الاصطناعي يخصص موقفًا محددًا لكل سيارة (عمود/صف/رقم).
  • إرشاد داخلي للموقف + تحقق آلي من الالتزام.
  • تطبيق مخالفات تلقائيًا على المخالفات الأساسية.

ثالثًا: لماذا هذا الحل فعّال؟

  • يمنع “العشوائية” من الأساس: لا أحد يبحث أو ينافس على موقف.
  • يوفّر عدالة: لكل سيارة موقف محدد، والمخالفات موثقة.
  • يرفع جودة الخدمة للجهة ويقلل الشكاوى والنزاعات.

رابعًا: ناتج المشروع للجهة

  • خفض الازدحام داخل المواقف.
  • حماية المواقف المخصصة ومنع التعديات.
  • تقارير تشغيلية تساعد في تحسين التخطيط واتخاذ القرار.
  • قابلية التوسع: Pilot ثم تعميم.
نقطة قوة رسمية: النظام “محايد” ويقلل الاحتكاك لأن القرارات آلية وواضحة: بطاقة ↔ لوحة ↔ موقف مخصص ↔ تحقق ↔ مخالفة.

1) المشكلة الحالية (مركّزة)

ازدحام وفقدان وقت

  • زحام داخل الممرات.
  • تكدس عند نقاط الالتقاء.
  • هدر وقت/وقود.

أخذ موقفين

  • انخفاض السعة الفعلية.
  • زيادة الضغط على بقية المستخدمين.
  • تكرار الشكاوى.

مواقف خاصة تُستغل

  • ذوي الإعاقة بدون تصريح.
  • كبار السن بدون صلاحية.
  • تولّد نزاعات.

2) جوهر الحل (قواعد إلزامية)

قاعدة (أ): دخول/خروج بالبطاقة فقط

  • تذكرة/بطاقة عند الدخول مرتبطة بلوحة السيارة.
  • لا فتح للبوابة إلا بعملية مسجلة.
  • الخروج أيضًا عبر البطاقة (أو تطابق اللوحة/العملية).
يمنع الفوضى من البداية

قاعدة (ب): تخصيص موقف إلزامي لكل سيارة

  • AI يحدد موقفًا واضحًا: A-07 / B-12.
  • الوقوف في غيره = تنبيه ثم مخالفة.
  • الموقف “محجوز منطقيًا” للسيارة حتى تُكمل الوقوف أو تُلغى العملية.
يوقف التنافس والنزاعات

قاعدة (ج): منع موقفين

  • تحليل تمركز السيارة داخل الخطوط عبر كاميرات.
  • تأكيد عبر حساسات المجاور.
  • تنبيه + مهلة قصيرة ثم مخالفة.
تطبيق آلي

قاعدة (د): حماية المواقف المخصصة

  • ذوي الإعاقة: تحقق من تصريح.
  • كبار السن/خاصة: صلاحيات حسب الجهة.
  • مخالفة فورية عند عدم وجود صلاحية.
أولوية قصوى

3) سير العمل (Workflow) — واضح للجهات

1) دخول

  • إصدار بطاقة.
  • ربط البطاقة بلوحة السيارة.
  • فتح البوابة.

2) تخصيص + إرشاد

  • رصد المواقف الفارغة.
  • تخصيص موقف إلزامي.
  • إرشاد داخلي للوصول.

3) تحقق + خروج

  • تأكد من الالتزام.
  • مخالفة عند المخالفة.
  • خروج بالبطاقة وتحديث الفراغ.
كيف يظهر رقم الموقف للسائق؟
  • شاشة عند المدخل تعرض: “موقفك: B-12”.
  • لوحات داخلية + أسهم/إضاءة للوصول.
  • اختياري: تطبيق جوال لإرشاد أدق.
متى تُسجل المخالفة؟
  • إذا أخذ موقفين.
  • إذا وقف في غير الموقف المخصص.
  • إذا استخدم مواقف ذوي الإعاقة/كبار السن بدون صلاحية.
  • توثيق: صورة + وقت + موقف + لوحة.

4) المخالفات والغرامات (موثقة + تلقائية)

تسجيل المخالفات آليًا لتقليل الاحتكاك، مع حفظ دليل واضح يقلل الاعتراضات.

المخالفة الرصد الإجراء غرامة مقترحة الخطورة
أخذ موقفين تمركز السيارة + حساسات مجاورة تنبيه + مهلة قصيرة ثم مخالفة 100 – 300 ريال متوسط
الوقوف في غير الموقف المخصص مقارنة الموقف المخصص بموقع السيارة تنبيه + إعادة توجيه / مخالفة 150 – 300 ريال متوسط
موقف ذوي الإعاقة بدون تصريح قراءة لوحة + قاعدة التصاريح مخالفة فورية + توثيق 500 – 1000 ريال عالي
موقف كبار السن/موقف خاص بدون صلاحية تعريف المنطقة + صلاحيات تنبيه أو مخالفة حسب سياسة الجهة 200 – 400 ريال متوسط
تلاعب بالبطاقة / محاولة دخول بدون تسجيل بوابات + تطابق اللوحة/العملية منع العملية + إحالة للإدارة حسب سياسة الجهة عالي
ملاحظة ربط: يمكن تكامل المخالفات مع أنظمة الجهة الداخلية أو حسب الصلاحيات مع الجهات المختصة، مع حماية الخصوصية.

5) المتطلبات الإلزامية (صياغة RFP جاهزة للجهات)

هذا القسم مكتوب كمتطلبات واضحة يمكن وضعها مباشرة في وثيقة طلب عروض (RFP).

أ) نطاق التشغيل (Operational Scope)

  • دخول/خروج إلزامي عبر بطاقة أو آلية تحقق معتمدة من الجهة.
  • تخصيص موقف إلزامي لكل سيارة عند الدخول.
  • حجز منطقي للموقف حتى يتم الوقوف أو انتهاء مهلة التخصيص.
  • إرشاد داخلي للموقف المخصص (شاشات/لوحات/إضاءة).

ب) الالتزام والمخالفات (Compliance & Enforcement)

  • رصد “أخذ موقفين” تلقائيًا مع تنبيه ثم مخالفة.
  • رصد “الوقوف في غير الموقف المخصص”.
  • التحقق من مواقف ذوي الإعاقة عبر قاعدة تصاريح.
  • توثيق المخالفة: صورة/وقت/رقم موقف/لوحة.

ج) الأجهزة والبنية (Hardware)

  • حساس لكل موقف (Occupancy Sensor).
  • كاميرات قراءة لوحات عند البوابات (ANPR) + كاميرات مراقبة مناطق.
  • بوابات دخول/خروج مع قارئ بطاقة/تذكرة.
  • شاشات إرشادية داخلية وعند المدخل.

د) البرمجيات والذكاء الاصطناعي (Software & AI)

  • محرك تخصيص (Allocation Engine) يقلل الازدحام ويمنع التكدس.
  • محرك كشف مخالفات (Violation Engine) قائم على القواعد + رؤية حاسوبية.
  • لوحة تحكم (Dashboard) للمشغل وتقارير دورية.
  • سجل تدقيق (Audit Log) لكل العمليات.

هـ) التكامل والبيانات (Integration & Data)

  • تكامل اختياري مع أنظمة التصاريح (ذوي الإعاقة/موظفين/زوار).
  • واجهات API للتقارير أو الربط الداخلي.
  • سياسات احتفاظ بالبيانات وفق اشتراطات الجهة.

و) الأمان والخصوصية (Security & Privacy)

  • تحديد صلاحيات المستخدمين (Roles) داخل لوحة التحكم.
  • تشفير البيانات أثناء النقل والتخزين حسب سياسات الجهة.
  • إخفاء/حماية بيانات التعريف في التقارير العامة.
مخرجات المورد (Deliverables): تركيب وتشغيل + تدريب + دليل تشغيل + خطة صيانة + تقارير أداء + دعم فني خلال فترة التجربة.

6) التصميم المعماري (Architecture) — شكل احترافي

المخطط التالي يوضح المكونات الرئيسية وكيف تتواصل مع بعضها: بوابات، حساسات، كاميرات، محرك AI، المخالفات، لوحة التحكم، والتكامل.

شرح سريع للمخطط

  • الأجهزة الميدانية: بوابات + حساسات + كاميرات.
  • المنصة: تخصيص موقف + كشف مخالفات + قواعد بيانات وسجل تدقيق.
  • الإدارة: لوحة تحكم وتقارير.
  • التكامل: تصاريح ذوي الإعاقة + أنظمة داخلية + إشعارات.

مخرجات فنية يمكن طلبها من المورد

  • مخطط شبكات واتصال (LAN/PoE/5G حسب الموقع).
  • خطة تركيب الحساسات وعددها حسب المواقف.
  • سياسات الاحتفاظ بالصور/البيانات ومراعاة الخصوصية.
  • اختبارات قبول (UAT) قبل التسليم.

7) خطة التنفيذ (Pilot → تعميم)

المرحلة 1: Pilot

  • موقع واحد عالي الازدحام.
  • تغطية جزئية بالحساسات + بوابات + لوحة تحكم.
  • مؤشرات: زمن البحث، عدد المخالفات، رضا المستخدمين.

المرحلة 2: تعميم داخل المنشأة

  • تغطية كاملة لكل المواقف.
  • تفعيل الإرشاد الداخلي بالكامل.
  • تشغيل المخالفات والسياسات النهائية.

المرحلة 3: معيار موحد

  • تحويله لمعيار/اشتراط للمشاريع الكبيرة.
  • توحيد سياسات المخالفات والتصاريح.
  • إمكانية منصة مركزية حسب قرار الجهة.

المرحلة 4: تحسين بالـ AI

  • توقع الذروة وتوزيع الحمل مسبقًا.
  • تحسين المسارات داخل الموقف.
  • تحسين التخطيط بناءً على البيانات.
مؤشرات قياس نجاح (KPIs) مقترحة: متوسط وقت العثور على موقف • نسبة أخذ موقفين • نسبة إشغال المواقف • عدد المخالفات المتكررة • رضا المستخدمين.
